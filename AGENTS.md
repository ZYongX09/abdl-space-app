## 开发约定

### 1. 防崩溃措施（任务追踪）

每个开发任务开始前，将任务内容写入 `.opencode/task-progress.md`，格式如下：

```markdown
# 任务进度追踪

## 当前任务
- **任务名称**：<描述>
- **开始时间**：<时间>
- **状态**：进行中 / 已完成 / 已中断

## 步骤
- [ ] 步骤 1：xxx
- [ ] 步骤 2：xxx
- [x] 步骤 3：xxx（已完成于 <时间>）

## 恢复点
如果任务中断，从这里继续：<描述下一步>
```

- 执行每一步前先写入步骤内容
- 每完成一步立即更新状态
- 任务完成后标记「已完成」
- 如果会话中断，下次读取此文件恢复进度

## 敏感操作（必须经过用户确认）

1. `wrangler deploy`（部署后端）
2. `wrangler secret put`（设置密钥）
3. 修改 `wrangler.jsonc` 配置
4. D1 数据库写操作（INSERT/UPDATE/DELETE）
5. GitHub 仓库设置变更
6. 任何涉及 API Key、Secret 的操作

> 注：Git 操作（commit、push 等）无需确认，可直接执行。

---

## 开发顺序（Phase 1）

1. 项目初始化：Gradle、依赖、主题系统
2. 通用组件：AppCard、AppButton、AppTextField、LoadingAnimation
3. 网络层：Retrofit API、Token 拦截器、错误处理
4. 认证模块：登录 → 注册 → 忘记密码 → Token 管理
5. Feed 页：帖子列表 + 发帖 + 图片上传 + 点赞 + 评论
6. 个人主页：我的主页 + 他人主页 + 关注/取关
7. 搜索：帖子和用户搜索
8. 通知：通知列表 + 已读
9. 纸尿裤评分：列表 → 详情 → 提交评分
10. 设置：主题切换、关于页
11. 深色模式适配
